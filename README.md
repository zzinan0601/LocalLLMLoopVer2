# local-llm-loop ver2

로컬 LLM(Ollama)으로 복합 질문을 서브질문으로 분해하고
각 서브질문을 순서대로 loop 판단하여 최종 답변을 합성하는 프로젝트

---

## 📁 프로젝트 구조

```
local-llm-loop-a/
│
├── main.py                     # 실행 진입점
├── requirements.txt            # 필요 패키지 목록
│
├── graph/
│   ├── state.py                # 상태 정의
│   ├── nodes.py                # 분해 / 답변 / 판단 / 다음 / 합성 노드
│   └── graph.py                # 그래프 조립 및 분기 설정
│
├── llm/
│   └── ollama_client.py        # Ollama LLM 연결 설정
│
└── prompts/
    └── templates.py            # 프롬프트 템플릿 모음
```

---

## ⚙️ 실행 흐름

```
질문 입력
    ↓
[분해 노드]  LLM이 서브질문으로 분해 + 순서 결정
    ↓
[답변 노드]  현재 서브질문 답변 생성 (이전 답변들 context로 포함)
    ↓
[판단 노드]  답변이 충분한가?
    ├─ NO  → 답변 노드로 되돌아감 (최대 3회)
    └─ YES → [다음 노드] 다음 서브질문 있나?
                ├─ 있음 → 답변 노드 (다음 서브질문 처리)
                └─ 없음 → [합성 노드] 전체 답변 하나로 합성
                               ↓
                           최종 답변 출력
```

---

## 🧩 노드 역할

| 노드 | 역할 |
|------|------|
| `decompose` | 질문을 서브질문 리스트로 분해, 의존 순서대로 정렬 |
| `answer` | 현재 서브질문에 답변 (이전 context 또는 이전 답변 참고) |
| `judge` | 답변 품질 판단 (GOOD / BAD) |
| `next` | 현재 답변 누적 후 다음 서브질문으로 이동 |
| `synthesize` | 모든 서브질문 답변을 하나의 완성된 답변으로 합성 |

---

## 🔀 분기 조건

| 분기 | 조건 | 이동 |
|------|------|------|
| `should_loop` | BAD 판정 | 답변 노드 (loop) |
| `should_loop` | GOOD 판정 | 다음 노드 |
| `should_continue` | 서브질문 남음 | 답변 노드 |
| `should_continue` | 서브질문 소진 | 합성 노드 |

## 💡 실행 예시

```
질문을 입력하세요: 전기차의 장단점과 미래 전망은?

==================================================

[분해 노드] 질문을 서브질문으로 분해 중...
[분해 노드] 서브질문 3개 생성:
  1. 전기차의 주요 장점은?
  2. 전기차의 단점과 한계는?
  3. 위 내용을 바탕으로 전기차의 미래 전망은?

[답변 생성 중...]
전기차의 주요 장점은... (실시간 출력)

[판단 노드] 결과: GOOD → 다음으로
[다음 노드] 1번 완료 → 2번

[답변 생성 중...]
전기차의 단점으로는... (실시간 출력)

[판단 노드] 결과: BAD → loop 재시도

[답변 생성 중...]
보완하자면, 전기차의 단점은... (개선된 내용 실시간 출력)

[판단 노드] 결과: GOOD → 다음으로
[다음 노드] 2번 완료 → 3번

[답변 생성 중...]
앞선 내용을 종합하면, 미래 전망은... (실시간 출력)

[판단 노드] 결과: GOOD → 다음으로
[다음 노드] 3번 완료 → 합성 단계

[최종 합성 중...]
전기차는 환경적 이점과... (실시간 출력)

==================================================
✅ 완료
```
